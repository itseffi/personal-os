# Challenging Stakeholder Questions

Generate probing questions that pressure-test a proposal.

## Instructions

### Step 1: Parse context
- Use goal/context and stakeholder details.
- State assumptions if details are missing.

### Step 2: Generate question set
- Produce at least 20 non-redundant questions.
- Cover: financials, resources, timeline, risks, market, competition, feasibility, UX, ethics, sustainability, team capability, performance history, strategic alignment, compliance, scalability.

### Step 3: Check quality
- Remove generic and repetitive questions.
- Ensure each question ties to provided context.

## Output

- Numbered question list (20+)

## When to Use

Use this skill when the task directly matches the workflow described above.

## When Not to Use

Do not use this skill when the request is unrelated, low-stakes, or better handled by a simpler direct response.
